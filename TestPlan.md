<a name="introduction"></a>
## 1. Введение
Данный документ описывает план тестирования веб-приложения **BookNest** — менеджера личной коллекции книг.  
Цель тестирования — проверка соответствия поведения приложения требованиям, изложенным в документе [SRS.md]([SRS.md](https://github.com/StasRimashewskii/BookNest/blob/cleanup/Requirements/SRS.md)).  
Тестирование охватывает весь ключевой функционал: аутентификацию, управление коллекцией, интеграцию с Google Books API, статусы чтения, поиск, фильтрацию и офлайн-режим через PWA.

---

<a name="items"></a>
## 2. Объект тестирования
Объектами тестирования являются следующие функциональные возможности:  
- Регистрация и аутентификация (email/пароль, Google OAuth)  
- Добавление, редактирование, удаление книг  
- Поиск книг через Google Books API по ISBN  
- Поиск, фильтрация и сортировка в личной коллекции  
- Просмотр детальной информации о книге  
- Управление статусами чтения  
- Офлайн-доступ (PWA): просмотр кэшированных книг, изменение статуса, синхронизация  

---

<a name="quality"></a>
## 3. Атрибуты качества
1. **Функциональность**  
   - Полнота: реализованы все функции из SRS  
   - Корректность: поведение соответствует требованиям  
2. **Удобство использования**  
   - Интуитивно понятный интерфейс  
   - Основные действия — не более 3 шагов  
3. **Надёжность**  
   - Сохранение данных при любых операциях  
   - Корректная синхронизация офлайн-изменений  
4. **Производительность**  
   - Загрузка коллекции (100+ книг) ≤ 1.5 сек  
   - Поиск в коллекции — мгновенно  

---

<a name="risk"></a>
## 4. Риски
- Потеря данных при сбоях сервера или ошибках синхронизации  
- Ошибки интеграции с Google Books API (таймауты, неверные данные)  
- Дублирование книг по ISBN  
- Некорректное кэширование в PWA (устаревшие или отсутствующие данные)  
- Проблемы адаптивности на мобильных устройствах  
- Медленная загрузка при большом объёме коллекции  

---

<a name="features"></a>
## 5. Аспекты тестирования

### 5.1 Регистрация и аутентификация
- Регистрация по email и паролю  
- Регистрация через Google OAuth  
- Вход и выход из аккаунта  
- Восстановление пароля  
- Обработка невалидных данных (email, слабый пароль)  

### 5.2 Управление коллекцией книг
- Добавление книги вручную  
- Добавление по ISBN с автозаполнением через API  
- Редактирование и удаление книги  
- Запрет дублирования по ISBN  

### 5.3 Поиск и фильтрация
- Поиск в коллекции по названию, автору, ISBN  
- Фильтрация по статусу чтения  
- Сортировка по дате, названию, автору  
- Сброс фильтров  
- Обработка пустого результата  

### 5.4 Просмотр информации о книге
- Отображение метаданных (обложка, описание, жанр)  
- Добавление и редактирование личных заметок  
- Изменение статуса чтения  

### 5.5 Управление статусами чтения
- Установка статусов: «Хочу прочитать», «Читаю», «Прочитал»  
- Отображение статуса в списке и на странице книги  
- Изменение статуса офлайн с последующей синхронизацией  

### 5.6 Работа офлайн (PWA)
- Установка PWA на устройство  
- Просмотр кэшированных книг без интернета  
- Изменение статуса офлайн  
- Синхронизация при восстановлении соединения  
- Блокировка добавления и поиска новых книг офлайн

<a name="nonfunctional"></a>
## 6. Нефункциональные требования
- Адаптивный интерфейс (десктоп, планшеты, мобильные)  
- Поддержка тёмной темы  
- Загрузка списка книг ≤ 1.5 сек  
- Поиск в коллекции — мгновенно  
- HTTPS везде  
- Защита от SQL-инъекций, XSS, CSRF  
- Rate limiting  
- Uptime ≥ 99.5%  
- Поддержка Chrome, Firefox, Safari, Edge (последние 2 версии)  
- PWA: кэширование обложек через CDN, опциональные push-уведомления  

---

<a name="approach"></a>
## 7. Подходы к тестированию
- **Тип**: ручное тестирование  
- **Инструменты**: веб-браузеры (Chrome, Firefox), DevTools, эмуляция мобильных устройств  
- **Офлайн-тесты**: отключение сети, работа с Service Worker  
- **Формат тест-кейса**:  
  - Предусловия  
  - Шаги воспроизведения  
  - Входные данные  
  - Ожидаемый результат
